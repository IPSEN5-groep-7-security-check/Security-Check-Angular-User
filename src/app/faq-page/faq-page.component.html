Het is buitengewoon moeilijk om een objectieve waarde toe te kennen aan een subjectieve vraag als "Hoe slecht is het niet implementeren van HTTP Strict Transport Security?" Dit wordt gecompliceerd door het feit dat wat voor de ene site misschien niet nodig is, zoals het implementeren van een inhoudsbeveiligingsbeleid, belangrijke risico's voor een andere site kan verkleinen.

De scores en cijfers die door het Mozilla Observatorium worden aangeboden, zijn ontworpen om ontwikkelaars te waarschuwen wanneer ze geen gebruik maken van de nieuwste webbeveiligingsfuncties, zoals aanbevolen in Mozilla's web security guidelines en server-side server side TLS guidelines. De huidige maximaal mogelijke score is 135 van de 100. De minimumscore is altijd 0, ongeacht hoe slecht een site het doet.

Voor technische details, zie grade.py, en om een lijst van de meest recente "perfecte" scorewebsites te zien, kunt u de getRecentScans API gebruiken.

Sommige tests kennen positieve scores toe, zoals +5 of +10 punten. Waarom? Zie het als het krijgen van extra krediet voor het verder gaan dan de plicht bij het verdedigen van uw website.
